# 论文技术路线详解 (GTG-Shapley)

GTG-Shapley 旨在解决传统 Shapley Value (SV) 在联邦学习中计算开销过大的问题。它的核心技术路线包含以下几个关键模块：

## 1. 消除子模型重训 (Elimination of Sub-Model Retraining)
* **核心思想**：为了避免在评估 SV 时对所有客户端组合进行从头训练，算法利用客户端上传的**梯度更新（Gradient Updates, $\Delta_i$）**来重构子模型。
* **重构公式**：
  给定上一轮的全局模型 $M^{(t)}$ 和客户端集合 $S$ 的更新，重构模型 $\tilde{M}_S^{(t+1)}$ 的评估为：
  $$V(S) = V(\tilde{M}_S) = V(M^{(t)} + \sum_{i \in S} \frac{|D_i|}{|D_S|} \Delta_i)$$
  这里不需要重新训练，只需按权重聚合子集 $S$ 的梯度即可。

## 2. 整体联邦学习贡献累加流程 (Algorithm 2 逻辑)
* 初始化所有客户端的总 SV 为 0：$\phi_i = 0$。
* 在每一轮联邦学习 $t$ 中：
  1. 客户端本地训练，上传梯度更新 $\Delta_i^{(t+1)}$。9k
  2. 服务器使用 `FedAvg` 聚合所有更新，得到 $M^{(t+1)}$。
  3. 服务器调用 `GTG-SHAPLEY` 算法计算该轮各客户端的贡献 $\phi_i^{(t+1)}$。
  4. 累加总贡献：$\phi_i = \phi_i + \phi_i^{(t+1)}$。

## 3. GTG-Shapley 核心算法实现 (Algorithm 1 逻辑)
在计算第 $t$ 轮的 SV 时，需要执行以下双层截断和引导采样策略：

### 3.1 轮间截断 (Between-round Truncation)
* **目的**：丢弃那些对模型性能提升不大的轮次。
* **逻辑**：计算当前轮的初始效用 $v_0 = V(M^{(t)})$ 和完全聚合后的效用 $v_N = V(M^{(t+1)})$。
* **判断**：如果 $|v_N - v_0| \le \epsilon_b$ （$\epsilon_b$ 为轮间截断阈值），则直接跳过本轮 SV 计算，所有客户端本轮 SV 记为 0。

### 3.2 引导蒙特卡洛采样 (Guided Sampling)
* **目的**：传统随机排列可能导致收敛慢，这里需要干预排列（Permutation $\pi^k$）的生成。
* **逻辑**：排列的前 $m$ 位（$m \ll n$）由 $n$ 个参与者固定循环轮替占据（确保每个人都有机会排在前面），剩下的 $n-m$ 位进行随机排列。

### 3.3 轮内截断 (Within-round Truncation)
* **目的**：在某个排列 $\pi^k$ 中，如果前面的客户端已经达到了这轮的极限性能，后面的客户端就不必再评估了。
* **逻辑**：遍历排列 $\pi^k$ 中的参与者 $j=1...n$。
* **判断**：设 $v_{j-1}^k$ 是加入前一个参与者后的模型效用。如果剩余潜在增益 $|v_N - v_{j-1}^k| \ge \epsilon_i$ （$\epsilon_i$ 为轮内截断阈值），则继续：
  1. 构建当前子集 $C = \{\pi^k[1], ..., \pi^k[j]\}$。
  2. 重构子模型 $\tilde{M}_C^{(t+1)}$ 并计算效用 $v_j^k = V(\tilde{M}_C^{(t+1)})$。
* **否则 (触发截断)**：
  停止重构和评估，直接让 $v_j^k = v_{j-1}^k$（这意味着边际增益为 0）。

### 3.4 Shapley Value 增量更新
* 对于排列 $\pi^k$ 中的第 $j$ 个参与者，其边际增益为 $(v_j^k - v_{j-1}^k)$。
* 结合之前的 $k-1$ 次采样，更新该参与者本轮的 SV：
  $$\phi_{\pi^k[j]}^{(t+1)} = \frac{k-1}{k}\phi_{\pi^k[j]}^{(t+1)} + \frac{1}{k}(v_j^k - v_{j-1}^k)$$

### 3.5 蒙特卡洛收敛判定 (Convergence Criterion)
* 循环执行排列采样直到满足收敛条件。收敛条件通常定义为估算的方差与均值的比例低于阈值，例如论文实验设置中使用了类似 $< 0.05$ 的判定标准。
