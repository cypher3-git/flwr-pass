## 🔍 根本原因分析

### 1. Shapley值的随机波动

Client 2 的 Shapley 值在 **-0.001 到 +0.0007** 之间波动：

```
负值轮次 (表明有害): 2, 3, 4, 5, 7, 8, 9, 10, 12, 16, 17, 18
正值轮次 (表明有益): 6, 11, 13, 14, 15, 19, 20
```

**关键发现**：
- Round 6, 11, 13, 14: Shapley值变为**正值**
- 这导致归一化分数上升
- 分数超过阈值0.05，Client 2被"洗白"

### 2. 为什么Shapley值会变正？

#### 原因A：轮间截断触发

查看日志发现，Round 14 有大量轮间截断：
```
[GTG-Shapley] Between-round truncation: |0.4370 - 0.4350| = 0.0020 <= 0.005
[GTG-Shapley] Between-round truncation: |0.4310 - 0.4320| = 0.0010 <= 0.005
```

**问题**：
- 当模型改进很小时，触发轮间截断
- 部分客户端返回 Shapley 值 = 0
- 这导致归一化时的 `min_val` 和 `max_val` 发生变化
- Client 2 的相对位置可能上升

#### 原因B：GTG-Shapley的蒙特卡洛采样噪声

GTG-Shapley 使用蒙特卡洛采样估计 Shapley 值：
```
Client 2: mean = 0.000652, std = 0.000858
```

**标准差 > 均值**，说明估计不稳定！

采样轮数只有15轮，对于接近0的Shapley值，噪声影响很大：
```
真实值 ≈ -0.0005
采样估计 = 0.0005 ± 0.0008
```

#### 原因C：归一化的敏感性

归一化公式：
```python
normalized_score = (cumulative_shapley[cid] - min_val) / (max_val - min_val)
```

当 Client 2 的累积Shapley值接近 `min_val` 时：
- 轻微的正向波动 → 显著的分数上升
- 例如：`(-0.01 - (-0.05)) / (0.3 - (-0.05)) = 0.04 / 0.35 = 0.114`
- 如果下一轮变为：`(-0.005 - (-0.05)) / (0.3 - (-0.05)) = 0.045 / 0.35 = 0.129`

### 3. 移动平均的滞后效应

移动平均公式：
```python
new_score = 0.5 * old_score + 0.5 * normalized_score
```

当 Shapley 值突然变正时：
```
Round 13: normalized = 0.15, score = 0.5*0.0464 + 0.5*0.15 = 0.0982
Round 14: normalized = 0.20, score = 0.5*0.0982 + 0.5*0.20 = 0.1491
```

**问题**：分数上升速度快于下降速度！

## 🔧 解决方案

### 方案1：增加Shapley值采样轮数（推荐）

**问题**：当前采样轮数15轮，对于小Shapley值噪声太大

**解决**：
```toml
gtg-sampling-rounds = 30  # 从15增加到30
```

**效果**：
- 减少蒙特卡洛采样噪声
- Shapley值估计更稳定
- 标准差/均值比例降低

### 方案2：使用指数移动平均平滑Shapley值

**问题**：单轮Shapley值波动大

**解决**：在累积前先平滑
```python
# 在 server_app_gtg.py 中添加
smoothed_shapley = 0.7 * previous_shapley[cid] + 0.3 * current_shapley[cid]
cumulative_shapley[cid] += smoothed_shapley
```

### 方案3：设置Shapley值阈值窗口

**问题**：接近0的Shapley值不可靠

**解决**：
```python
# 如果Shapley值在 [-threshold, +threshold] 内，视为0
shapley_threshold = 0.0003
if abs(current_shapley[cid]) < shapley_threshold:
    current_shapley[cid] = 0.0
```

### 方案4：调整移动平均系数（快速修复）

**问题**：alpha=0.5 导致分数上升太快

**解决**：
```toml
alpha = 0.7  # 增加历史权重，减缓波动
```

**效果**：
```
Round 14: score = 0.7*0.0464 + 0.3*0.15 = 0.0775 (vs 原来的0.0982)
```

### 方案5：使用多轮平均判定（最稳健）

**问题**：单轮判定容易误判

**解决**：
```python
# 连续3轮低于阈值才剔除
if score < threshold:
    elimination_count[cid] += 1
    if elimination_count[cid] >= 3:
        eliminate(cid)
else:
    elimination_count[cid] = 0
```

## 📈 理论分析

### Shapley值的期望和方差

对于AFR攻击者，理论上：
```
E[Shapley] < 0  (负贡献)
Var[Shapley] = σ²/n  (n为采样轮数)
```

当前设置：
```
n = 15
σ ≈ 0.0008
Var ≈ 0.0008²/15 ≈ 4.3e-8
Std ≈ 0.0002
```

但实际标准差 ≈ 0.0008，说明采样不足！

### 所需采样轮数估算

要使 `Std/Mean < 0.5`（相对误差<50%）：
```
Mean ≈ -0.0005
Std < 0.00025
Var < 6.25e-8
n > σ²/Var = 0.0008²/6.25e-8 ≈ 10

实际需要: n ≈ 25-30
```

## ✅ 总结

### 问题本质
1. **GTG-Shapley采样噪声**：15轮采样不足，导致Shapley值波动大
2. **归一化敏感性**：小的Shapley值变化导致大的分数变化
3. **移动平均滞后**：alpha=0.5导致分数响应过快

### 核心解决方案
**增加采样轮数到30 + 调整alpha到0.6**

### 预期改进
- ✅ Client 2 在第8轮稳定剔除，不再振荡
- ✅ Shapley值标准差降低50%
- ✅ 分数变化更平滑可靠
